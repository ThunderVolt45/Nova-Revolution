# 게임 디자인 설계 (Game Design Document)

## 1. 핵심 시스템 (Core Systems)

### 1.1 유닛 조립 시스템 (Unit Assembly System)
플레이어는 다양한 부품을 조합하여 자신만의 커스텀 유닛을 제작할 수 있습니다.
- **필수 부품:** 다리(Legs), 몸통(Body), 무기(Weapon)
- **선택 부품:** 액세서리(Accessory)
- **조립 방식:** 각 부품의 능력치가 합산되어 최종 유닛의 성능이 결정됩니다.
- **덱:** 조립된 유닛은 "덱" 에 등록되며 최대 10개의 유닛을 등록할 수 있음.

### 1.2 유닛 속성 (Unit Attributes)
모든 유닛은 다음의 속성을 가지며, 이는 부품들의 속성값 합계로 계산됩니다.
- **와트 (Watt):** 유닛 생산 시 소모되는 비용
- **체력 (HP):** 유닛의 생존력
- **공격력 (Attack):** 적에게 입히는 데미지
- **방어력 (Defense):** 받는 데미지 감소율
- **속도 (Speed):** 이동 속도
- **연사 (Fire Rate):** 공격 간격 (초당 공격 횟수 등)
- **시야 (Sight):** 안개를 밝히는 범위
- **사거리 (Range):** 최대 공격 가능 거리
- **최소 사거리 (Min Range):** 공격 불가 근접 거리
- **스플래시 범위 (Splash Range):** 폭발형 무기 등의 광역 데미지 범위

### 1.3 자원 시스템 (Resource System)
게임 내 자원은 시간에 따라 자동으로 충전되는 방식을 채택합니다.
- **와트 (Watt):** 유닛 생산 및 스킬 사용에 필요한 기본 자원
- **스킬 포인트 (SP):** 특수 스킬 사용에 필요한 자원

### 1.4 생산 시스템 (Production System)
- 미리 조립되어 덱에 등록된 유닛을 기지에서 즉시 생산합니다.
- 생산 시 해당 유닛의 총 와트 비용만큼 현재 보유 자원에서 차감됩니다.

### 1.5 스킬 시스템 (Skill System)
- 사령관(플레이어)은 와트와 SP를 소모하여 전황을 바꿀 수 있는 스킬을 사용합니다.
- **예시:** 기지 회복(Base Repair), 유닛 회복(Unit Heal) 등

### 1.6 인구수 시스템 (Population & Capacity)
유닛 생산은 다음 두 가지 제한 사항을 동시에 적용받습니다.
1. **와트 총합 제한:** 현재 전장에 배치된 유닛들의 와트 합계가 일정 수치를 넘을 수 없습니다.
2. **개체 수 제한:** 생산된 유닛의 총 마릿수가 제한 수치를 넘을 수 없습니다.
- **결과:** 두 제한 중 어느 하나라도 도달할 경우 추가 생산이 불가능합니다.

### 1.7 전장의 안개 (Fog of War)
노바 1492 방식의 2단계 안개 시스템을 채택합니다.
- **가시 구역 (Visible):** 아군 유닛의 시야(Sight) 범위 내 지역. 적 유닛, 건물, 투사체 등 모든 정보가 실시간으로 표시됩니다.
- **비가시 구역 (Foggy/Shroud):** 아군 시야가 없는 지역. 지형은 항상 보이지만, 적 유닛이나 건물 등은 가려지며 화면이 어둡게 처리됩니다.
- **미니맵 연동:** 미니맵 상에서도 아군 시야가 닿지 않는 곳은 어둡게 표시되어 전장 상황을 직관적으로 파악할 수 있게 합니다.

## 2. 게임 모드 (Game Modes)

### 2.1 공통 규칙
- 각 플레이어는 단 하나의 기지와 약간의 와트, SP를 가지고 게임을 시작하며 추가 건물을 건설할 수 없습니다.

### 2.2 AI 1vs1 대전 (AI 1vs1 Match)
- 플레이어는 AI와 1대1 대전을 진행할 수 있습니다.
- 기본적인 RTS 룰을 따르며, 적의 기지를 파괴하는 것이 목표입니다.

## 3. 승리 및 패배 조건 (Victory & Defeat Conditions)

### 3.1 승리 (Victory)
- 적의 기지를 파괴하면 게임에서 승리합니다.

### 3.2 패배 (Defeat)
- 플레이어의 기지가 파괴되면 게임에서 패배합니다.

## 4. UI/UX 가이드라인 (UI/UX Guidelines)

### 4.1 로비 및 조립 (Out-game)
- **3D 유닛 프리뷰:** 부품 교체 시 실시간으로 외형 변화를 확인할 수 있는 프리뷰 제공
- **능력치 표시:** 현재 조립 중인 조합의 능력치 값 (와트, 공격력 등) 을 보여주는 UI 제공
- **부품 필터링:** 부품 목록을 다리, 몸통, 무기, 악세사리로 구분해서 필터링해 표시
- **유닛 덱:** 현재 덱에 등록되어 있는 유닛들의 외형을 볼 수 있는 프리뷰 제공. 슬롯 선택시 해당 슬롯에 유닛을 조립/수정/해체 할 수 있음

### 4.2 인게임 인터페이스 (In-game HUD)
- **자원 정보창:** 실시간 와트, SP, 인구수(와트 총합 및 개체 수) 표시
- **유닛 생산 패널:** 미리 조립된 유닛 프리셋을 즉시 생산할 수 있는 단축키 슬롯 UI
- **상태창:** 선택된 유닛의 외형과 현재 체력, 상세 능력치, 버프/디버프 상태 표시
- **미니맵:** 전장 상황, 안개(Fog of War), 아군/적군 위치 시각화

### 4.3 조작 및 피드백 (Controls & Feedback)
- **RTS 표준 조작**
  - **기본 조작:** 왼쪽 클릭으로 유닛 선택, 마우스 드래그로 여러 유닛 선택, 오른쪽 클릭으로 목표 지정 등 RTS 표준 조작 체계를 가져감
  - **단축키:**
    - 부대 지정 (Ctrl + 1~0): 현재 선택된 유닛을 부대 지정
    - 부대 선택 (1~0): 저장된 부대를 선택. 연타 시 해당 부대의 위치로 카메라 이동.
    - 스마트 공격 (A -> 좌클릭): 유닛을 선택할 시 해당 유닛을 공격. 위치를 선택할 시 해당 위치로 이동하면서 이동 경로 내의 적을 자동 공격.
    - 정지 (S): 현재 수행 중인 명령 취소
    - 홀드 (H): 제자리에서 이동하지 않고 사거리 내 적만 공격
    - 순찰 (P): 현재 위치와 지정한 위치를 왕복. 이동 중 적을 만날 경우 자동으로 공격.
  - **스마트 커서:** 기본 커서, 목표 지정 커서 등 현재 수행하려는 액션에 따라 커서의 외형 변경
- **노바 1492 고유 조작**
  - **스마트 부대 지정:** 별도로 부대 지정을 하지 않은 경우 숫자 키 입력 시 조립된 유닛 덱에서 입력 숫자에 해당하는 유닛을 자동으로 선택. 별도로 부대 지정 시 부대 지정된 유닛들을 자동으로 선택. 부대가 해체되거나 소속 유닛이 전멸할 경우 기본 값으로 되돌아감.
  - **단축키:**
    - 유닛 생산 (Shift + 1~0): 입력 숫자에 해당하는 유닛을 즉시 생산
    - 스킬 발동 (Alt + 1~0 -> 좌클릭): 입력 숫자에 해당하는 스킬을 선택. 좌클릭으로 목표를 지정해 발동하거나 우클릭으로 취소. 
    - 유닛 산개 (C -> 좌클릭): 현재 선택된 유닛들을 해당 위치로부터 최소 사거리 만큼 이동
    - 유닛 정지 (L): 현재 선택된 유닛을 완전히 정지시킴
    - 기지 선택 (B): 플레이어의 기지를 자동으로 선택. 연타시 기지 위치로 카메라 이동.
- **경고 시스템:** 아군 공격 시 미니맵 알림 및 자원 부족 시 시각적/음향적 피드백 제공
